Week 5 Notes

Chapter 10: Testing and Debugging

  • exceptions returns a value which you can use program an error.
  • stack traces are a group of errors.
  • warnings are errors in the code but do not prevent the project from running and compiling.
  • 'use strict'; is a statement added at the top of the file that can be used to detect errors and classify sloppy code as errors.
  • it does not need to be at the top of the file but can be used as a "as needed" basis. For instance, inside functions.
  • we can use alert() as breakpoints in the code because it stops the code until "OK" is clicked.
  • you can use console.log() as well but just know it does not stop the code.
  • you can use web browsers like chrome to troubleshoot code. For instance debugger can be placed in functions to show what varibles contain and so forth.
  • make sure to remove debugger from the code before production otherwise the program will appear to freeze.
  • You can use try and catch methods (something you are already familiar on).
  • You can use test frameworks like jest to test your code.
